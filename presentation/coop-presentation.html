Execution modes

Safety + architecture posture

Local-first by default. Bounded execution when privileged.

Runtime principles

  • Thin runtimes, strong shared contracts
  • Explicit policy + approval + replay protection
  • Mock/live modes surfaced in UI
  • Arbitrum + Sepolia scoped chain posture

Bounded session-key slice

  • Green Goods garden actions only in phase-1 scope
  • Disallowed actions blocked before send
  • Revoked/expired/exhausted sessions rejected
  • Operator logs remain demo-legible

Coop avoids open-ended autonomous execution in v1 and prioritizes deterministic, auditable action paths.
