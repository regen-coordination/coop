1

First-time Setup

From install to first coop in 5 minutes
1
User

Install Extension

User installs Coop from Chrome Web Store. Extension shows popup with "Create a Coop" and "Join with invite" options.

2
User

Create Passkey Identity

User creates local identity using device passkey (Face ID, Touch ID, or PIN). No wallet extension needed.

3
User

Setup Ritual (4 Lenses)

User answers: What's the purpose? Who's involved? What are we tracking? What's the review cadence?

4
Extension

Coop Created + Safe Deployed

Extension creates coop state and (in live mode) deploys Safe on Arbitrum/Sepolia. Shows success with coop address.

5
Extension

Invite Generation

Creator generates trusted member or regular member invite. QR code and deep link created for sharing.

Flow Visualization

Install → Passkey → 4 Lenses → Safe Deploy → Coop Live
2

Member Join Flow

Joining an existing coop from invite
1
User

Receive Invite

User receives invite link via chat, email, or QR scan from coop creator. Link contains coop ID and invite token.

2
Extension

Accept Invite

Extension validates invite, shows coop details (name, purpose, members), asks user to confirm join.

3
Extension

State Bootstrap

Extension syncs coop state from peers via Yjs. New member sees existing shared memory and feed.

4
Extension

Ready to Participate

Member can now capture, review, publish, and see live updates from other members.

3

Capture & Review Flow

From loose tabs to published memory
1
User

Manual Round-up

User clicks "Round up now" in extension popup. Extension scans open tabs and extracts content.

2
Extension

Local Processing

Extension shapes extracts into candidates with heuristics: title, summary, domain, relevance score. All local.

3
User

Review in Roost

User reviews draft cards in sidepanel. Can edit title, summary, add tags, adjust relevance, or discard.

4
User

Explicit Publish

User clicks "Publish to Feed" — only now does content enter shared coop state. Privacy boundary respected.

5
Extension

Live Sync

Published artifact syncs to all online members via Yjs. Feed updates in real-time across all devices.

🔒 Privacy Boundary

Observation and shaping happen locally. Nothing enters shared state until human explicitly pushes it. This is the core privacy promise.

4

Receiver Pairing & Capture

Mobile capture into extension inbox

Extension Operator

1
Extension

Generate Pairing

In Coops tab, operator generates receiver pairing with QR code and deeplink.

2
Extension

Wait for Intake

Extension monitors member-private intake queue for synced captures.

3
Extension

Convert & Publish

Operator converts intake items to drafts, reviews, and publishes to Feed.

Mobile Receiver

1
Receiver

Scan QR / Open Link

User opens /pair on mobile PWA, scans QR or accepts deeplink payload.

2
User

Capture Voice/Photo

User taps egg button to record voice, takes photo, or attaches file.

3
Receiver

Sync to Extension

Capture queues locally, syncs to extension when online. Shows in inbox.

5

Archive & Export Flow

Preserving memory with Filecoin receipts
1
User

Select Archive Target

User chooses: artifact capsule, coop snapshot, or memory seed bundle from Feed or Board.

2
Extension

Prepare Bundle

Extension compiles structured JSON + metadata. In mock mode: simulates. In live mode: delegates to trusted node.

3
Extension

Storacha Upload

Bundle uploads via delegated Storacha space. Returns root CID and gateway URL.

4
Extension

Receipt Generated

Archive receipt saved locally: timestamp, CID, gateway URL, purpose. Visible in archive log.

5
User

Export or Share

User exports receipt bundle (JSON) or shares gateway link. Filecoin deal follows async.

6

Weekly Review Ritual

Group synthesis and next-step planning
1
User

Gather Context

Members open Flock Meeting tab. See: private finds, working drafts, ready-to-share items, and board snapshot.

2
User

Review Together

Grouped view by category and member. Discuss, combine, refine. Move items to "ready" or archive.

3
User

Publish Batch

Multiple drafts published to Feed. Board updates with new artifacts. Members see live updates.

4
Extension

Snapshot & Archive

Post-meeting snapshot created. Optional: archive to Filecoin for durable weekly memory.

🎯 Demo-Critical Paths

For hackathon demo, focus on Flows 1→2→3: Create coop → Join with peer → Round up tabs → Review → Publish → See sync. Add Flow 4 (receiver pairing) if time permits.
