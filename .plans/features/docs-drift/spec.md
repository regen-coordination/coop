# Docs Drift Maintenance

**Feature**: `docs-drift`
**Status**: Active
**Source Branch**: `chore/docs-drift`
**Created**: `2026-03-26`
**Last Updated**: `2026-03-26`

## Summary

Continuously keep docs aligned with the real codebase so commands, architecture notes, plan locations, and user-facing explanations do not drift or hallucinate.

## In Scope

- repo docs and instructions drift
- stale commands and validation references
- mismatches between plans, docs, and code
- missing updates when product surfaces materially change

## Out Of Scope

- writing net-new marketing docs with no code change trigger
