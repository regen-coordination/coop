## High Findings

### H1. Missing `SetupInsightsInput` type export from `@coop/shared/app` [NEW]
- **File**: `packages/shared/src/app-entry.ts:8-11`
- **Issue**: `SetupInsightsInput` interface is defined at `shared/modules/coop/setup-insights.ts:3` but `app-entry.ts` only re-exports the value `emptySetupInsightsInput`, not the type. Three app files import `type { SetupInsightsInput } from '@coop/shared/app'`:
  - `packages/app/src/views/Landing/index.tsx:1` (TS2724)
  - `packages/app/src/views/Landing/landing-data.ts:1` (TS2724 + cascading TS2538, TS7053)
  - `packages/app/src/views/Landing/landing-types.ts:1` (TS2724)
- **Impact**: All 7 type errors in the app package stem from this single missing export. The Landing page works at runtime (Vite resolves it through the `emptySetupInsightsInput` value's inferred type) but TypeScript compilation fails.
- **Implicit `any` cascade**: Without the type, parameters in `index.tsx:171` and `index.tsx:707` gain implicit `any` types, and `landing-data.ts:415,418` lose index-type safety.
- **Recommendation**: Add `type SetupInsightsInput,` to the re-export block in `app-entry.ts:8-11`.
