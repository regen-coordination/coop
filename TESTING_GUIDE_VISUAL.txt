WHEN YOU FIND A BUG
════════════════════════════════════════════════════════════════════════════════

    1. Open TESTING_ISSUES.md
    2. Copy the issue template
    3. Fill in:
       • What flow you're on
       • Steps to reproduce
       • Expected behavior
       • Actual behavior
       • Console error (if any)
       • Severity: blocker | major | minor | polish

    Example:

       ### Flow 2 – Coop creation fails

       **Component:** extension
       **Severity:** major

       **Steps to Reproduce:**
       1. Click "Create Coop"
       2. Pick "Friends" preset
       3. Enter name
       4. Click Create

       **Expected:**
       Coop appears in list

       **Actual:**
       Error: "Failed to create"

       **Console Error:**
       TypeError: name is undefined

TROUBLESHOOTING QUICK LOOKUP
════════════════════════════════════════════════════════════════════════════════

Problem: "Extension won't load"
  → Check .env.local exists at repo root
  → Kill dev:extension, restart it
  → Delete packages/extension/dist, rebuild

Problem: "Can't pair with receiver"
  → Check bun run dev:api is running (Tab 3)
  → Verify signaling URL in Settings

Problem: "No sync between profiles"
  → Check both profiles have SAME signaling URL in .env.local
  → Restart all 3 servers
  → Reload extension in chrome://extensions

Problem: "Console shows errors"
  → Screenshot the error
  → Note the file and line number
  → Add to issue report

Problem: "Config change not applied"
  → Edit .env.local
  → Kill dev:extension (Ctrl+C)
  → Restart: bun run dev:extension
  → Reload extension in Chrome
