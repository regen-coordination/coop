# Coop Demo Storyboard (90s / 5min / 12min)

**Version:** v0  
**Date:** 2026-03-08  
**Audience:** judges, funders, ecosystem partners, pilot communities

---

## 0) Demo prep baseline (all versions)

- Anchor running (`@coop/anchor`) and reachable on `:8787` / `:8788`.
- Extension loaded and sidepanel pinned.
- PWA open on mobile (or responsive viewport fallback).
- One coop pre-created OR fresh-create script ready.
- Backup captures ready:
  - 1 web article tab
  - 1 voice transcript fallback text
  - 1 dropped file (txt/md)

---

## A) 90-second lightning demo

## Objective
Prove the core loop in under 90 seconds: **capture → AI structure → collaborative context**.

### Scene breakdown

| Scene | Time | Intent | On-screen action |
|---|---:|---|---|
| A1 Hook | 0:00-0:10 | Set problem | “Community work is fragmented across tabs, notes, and calls.” |
| A2 Coop creation/join | 0:10-0:25 | Show no-login onboarding | Create or join coop via share code |
| A3 Capture | 0:25-0:45 | Show frictionless input | Capture current tab or quick voice note |
| A4 Process | 0:45-1:10 | Show AI value | Run skill (impact-reporting), show summary + actions |
| A5 Close | 1:10-1:30 | Show why it matters | “Now evidence is structured and shared in real time.” |

### Talk track (short script)
- “Coop is a browser-native knowledge commons for local coordination.”
- “No account required: create or join with one share code.”
- “Capture from where work already happens—voice and tabs.”
- “One click runs a pillar skill and turns raw notes into actionable output.”
- “This gives communities a living evidence graph, not scattered chat fragments.”

### Fallback notes
- If voice mic fails: paste prepared transcript and process it.
- If AI request fails: show baseline extraction and narrate fallback.
- If websocket is unstable: refresh feed and call out local persistence.

## B) 5-minute product walkthrough

## Objective
Demonstrate complete workflow and reliability signals, including multi-surface UX.

### Scene breakdown

| Scene | Time | Intent | On-screen action |
|---|---:|---|---|
| B1 Context | 0:00-0:30 | Why now | Problem + Coop thesis |
| B2 Onboarding | 0:30-1:15 | Easy setup | Create coop + show share code |
| B3 Multi-input capture | 1:15-2:20 | Breadth of capture | Capture tab + voice + drag/drop |
| B4 Skill processing | 2:20-3:20 | Structured output | Process 1-2 items with pillar switch |
| B5 Canvas bridge | 3:20-4:20 | Sensemaking | Open canvas and connect capture to insight |
| B6 Mobile reliability | 4:20-4:50 | Field use case | Show PWA offline queue/status |
| B7 Wrap | 4:50-5:00 | Ask / next step | Pilot invitation + roadmap |

### Talk track (script blocks)

**B1**
- “Most community coordination tools optimize chat, not usable evidence.”

**B2**
- “In Coop, teams form instantly with a share code—no signup flow.”

**B3**
- “Capture is modality-agnostic: web context, voice, and files all land in one feed.”

**B4**
- “Skills structure this into summary, metrics, and immediate next actions.”

**B5**
- “Canvas turns feed events into a map: what happened, why it matters, what to do.”

**B6**
- “Mobile supports offline capture with queue + sync so fieldwork doesn’t get lost.”

**B7**
- “We’re ready for pilots focused on impact reporting and coordination.”

### Fallback notes

## C) 12-minute deep demo (technical + product)

## Objective
Show end-to-end architecture value: collaboration, skill pillars, data reliability, and UX trajectory.

### Scene breakdown

| Scene | Time | Intent | On-screen action |
|---|---:|---|---|
| C1 Problem + positioning | 0:00-1:00 | Strategic framing | Pitch frame: browser-first commons |
| C2 Fast onboarding | 1:00-2:00 | Zero-friction start | Create + join in two identities |
| C3 Real-time collaboration | 2:00-4:00 | Multi-user sync | Simultaneous captures in two clients |
| C4 Skill pillar comparison | 4:00-6:30 | Explain differentiation | Run same content through 2 pillars |
| C5 Canvas sensemaking | 6:30-8:30 | From feed to graph | Map captures, link insights, inspect actions |
| C6 Reliability path | 8:30-10:00 | Operational trust | Offline queue, reconnect sync, status signaling |
| C7 UX polish delta | 10:00-11:15 | Current vs target | Show storyboard visuals + planned polish improvements |
| C8 Close + Q&A setup | 11:15-12:00 | Clear next step | Pilot criteria, integration trajectory |

### Deep-demo script prompts

- “Here is one workflow in two clients to prove real-time membrane behavior.”
- “Notice how pillar choice changes extracted structure and recommended actions.”
- “Canvas is where fragmented captures become an operational map.”
- “Offline queue + local persistence keep capture resilient in real settings.”
- “Next polish phase upgrades clarity and trust without changing core architecture.”

### Fallback notes
- If second client unavailable: use prerecorded clip/GIF for sync proof.
- If voice recognition inconsistent: use tab + file captures as primary path.
- If skill endpoint throttles: show prior output JSON/card screenshot and continue.
- Keep a final static “summary slide” to land value even after live glitches.

## 1) Stagecraft checklist (for all demo lengths)

- Keep cursor still while speaking; avoid frantic scrolling.
- Zoom browser at 110-125% for projector readability.
- Use named coop and realistic transcript (not lorem ipsum).
- Narrate states explicitly: “processing,” “completed,” “queued offline.”
- End with one concrete CTA (pilot, design partner, or infrastructure partner).

---

## 2) Optional one-line closers

- “Coop turns scattered community activity into structured, shared intelligence.”
- “This is not just capture—it’s capture that immediately becomes coordination.”
- “From field note to collective action in one loop.”
